# Paper release notes

Use this file to keep the repository and manuscript synchronized.

## Put these files under `paper/`

At minimum:

- `paper/manuscript.pdf`
- `paper/source.zip` or `paper/arxiv.tex` + any required source tree
- `paper/figures/` or a figure-source archive if you want exact paper reproducibility inside the repo

## What to record here

- the manuscript title currently used in the repository;
- the commit hash that matches the submitted or posted manuscript;
- any differences between manuscript numbers and the public `metrics.json` files;
- whether large artifacts are attached to a GitHub Release instead of the git tree.

- Manuscript title:
- Commit hash:
- PDF filename:
- Source filename:
- Release tag / DOI:
- Known repo-manuscript mismatches:
